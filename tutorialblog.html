Introduction:

Hello! I am Hannah, and today we will be going over positioning data in rows and columns using HTML and CSS. This is an absolute beginner friendly tutorial, so if you’re new to web development, don’t worry, I’ll walk you through every step of the way.

HTML stands for Hypertext Markup Language, and it serves as the foundation of a webpage by defining its structure and content. This includes text, images, links, and various elements that make up a site. However, while HTML provides the building blocks, it’s CSS—Cascading Style Sheets—that brings a webpage to life by making it visually appealing and well-organized. Today, we’ll be focusing on how CSS helps structure web pages by properly aligning elements like paragraphs, headers, and navigation bars.

There are multiple ways to organize and align text and other elements in CSS, but in this tutorial, we will focus on using CSS Grid. CSS Grid is a powerful layout system that allows you to create complex, responsive designs with ease. It allows you to arrange content into rows and columns with ease. It is truly a great beginner skill and transfers for more advanced web programming.

The final product in this tutorial will be a rudimentary website that has a navigation bar, a side menu, and a section for text. These will each be created using the CSS Grid and contain text. On to more coding! We will start off by building our content, so head on over to the index.html file.

Tutorial:

Step 1:

For our first step, we will need to create an index.html file and a styles.css file. With these two files, we will be able to explore the grid functionality in CSS. Create these files in the folder that you want to create your website in

Step 2:

Go to the index.html file, there are a couple of prerequisites for every HTML file. You need to define the doctype, a header, and a body.

Typical syntax looks like this: HTML Online Editor

<!DOCTYPE html>

<html>

    <head></head>

<title>CSS Grid Tutorial</title>

<body>

<p>This is some text.</p>

</body>

</html>

Step 3: 

Link the CSS file to the HTML file

Before we get any further, we will need to link the CSS file to the HTML file. This should be done in the head of the HTML file.

This will be done by adding, “<link rel="stylesheet" href="styles.css">” between the <head> elements. This is called external CSS and makes it so all of the stylization happens dynamically from a different file.

Step 4:

Another preparatory step before we get to coding is using a live server. Install the Live Server extension created by Ritwick Dey on VS Code. Doing so will allow you to have this additional functionality to check the website that you are creating.

Once the Live Server is installed, click, “Go Live” in the bottom right side of your VS Code window. From here, it will take you to your favorite web browser and display the website you have made so far.

Step 5:

Create a div element with class= “container” in index.html.

Create three elements between the div header and footer where the class of each item is, nav-bar, side-menu, and body-text.  

This should look like:

<div class="container">

        <div class="item nav-bar">nav-bar</div>

        <div class="item side-menu">side-menu</div>

        <div class="item body-text">body text</div>

    </div>

For more information on classes and elements go here:

We will be manipulating these three objects to create a full layout of the contents we want to put into our website.

Aside:

In CSS and HTML, there are two types of elements: inline and block. As you can see in the image above, block elements stack vertically. Inline elements flow the content from one line to the next. While we are working on our Grid CSS, we are using block elements. This is because they are statically placed instead of dynamically fitting. We have to define where the elements are instead of them flowing through our web page.

Step 5:

Go to CSS page and define the .container class and with:

.container {

    display:grid;

    grid-template-rows: 100px 100px 100px 100px 100px 100px;

    grid-template-columns: 100px 100px 100px 100px 100px 100px;}

 This code here is creating the container class used in the div element on the index.html page. Its functionality is creating a grid where each row and column is 100 pixels and there are six of them.

We can now look at our Live Server and see what is happening with our code and this new grid class.

Now when you inspect your website and hover on the <div class=”container”> line, it displays the grid that is happening in the background. Now that we have created the grid element, we should add stylization to the items so we can better see them displayed on the screen.

Step 6:

In this step we will add background colors to the div items, such as nav-bar and side-menu so that we can visually see the different grid levels. To do this step, we need to add classes for each element to our styles.css file.

.nav-bar {

    background-color: lightblue;

}

.side-menu {

    background-color: lightgreen;

}

.body-text {

    background-color: lightyellow;

}

Now when you go back to your live server, you should be able to see colored boxes around each of these elements.

Thought Question:

Did you keep the initial “<p>This is some text.</p>”?

Is the text, “This is some text.” go way down below the grid that you have created with your small three objects?

The way that grid containers work in CSS allocate and reserve space that is defined. Since there were six rows of 100px and six columns of 100px, that space is all allocated for the elements in the div. Hence why the “This is some text” is far down at the bottom of the page.

Step 6:

Now that we can visually see the changes that we are making to our elements, let's lay them out in different ways. One way is using grid-column and grid-row for each element class. Since we are creating a navigation bar, we need to extend the lengths of each item for our nav-bar element.

Our grid starts its index at 1 and goes until 7. If we want to cover the entire top bar, we will want our nav-bar column to go from 1/7 and row 1/2.

The syntax for this part looks like:

.nav-bar {

    background-color: lightblue;

    grid-row: 1 / 2;

    grid-column: 1 / 7;

}

For the side-menu element, we should have grid-row be 2/7 and grid-column be 1/2.

For the body-text element, we should have grid-row be 2/7 and grid-column be 2/7.

Step 7:

Now that you have your grid elements set up, it is important to test the limits and extents of the areas we defined.

Take a moment to overlap the grids.

Take a moment to choose an index that is greater than 7.

Step 8:

In the previous step, we learned that the layers overlap and this is based on the order of initialization. To define this specifically, we need to use the z-index. This z-index is the order of the elements in the grid. Change the z-index number to 1 to move the element object to the front.

.nav-bar {

    z-index: 1;

}

Step 9:

While grid-row and grid-column are a good way to do this, we can also use the class method, grid-area. Grid area works like:

.nav-bar {

        grid-area:row-start col-start row-end col-end;

}

Take the time to replace each of your elements to use grid-area instead of grid-row and grid-column. This is for further readability and simplicity in the code. Nothing on your live server should change in terms of content, but the code that created it should.

Oftentimes in web design, the end result is the same or extremely similar. The way that the backend is written, however, can be wildly different.

Conclusion:
